2020-02-28 算法复杂度

算法复杂度大致可分为两部分：时间复杂度 和 空间复杂度 。（当然还有其它一些影响不大的因素，在此忽略）

Case 1. 时间复杂度

一些术语名词

关于时间复杂度，我们有一些定义。必须 知道的有：

1.所谓时间复杂度其实是一个模糊的概念，它指一个函数，描述某算法的运行时间。并常用 大O符号 表述。

2.我们用 T(n)T(n)T(n) 表示一个算法的 最坏复杂度 。并且 一般来说，nnn 为数据规模。当然 T(n)T(n)T(n) 也可以表示 时间频度 。时间频度 指一个算法中的语句执行次数，不忽略低阶项和常数项。 另外的，如果 T(n)=O(x)T(n)=O(x)T(n)=O(x) 且 xxx 为常数，则我们认为这等同于： T(n)=O(1)T(n)=O(1)T(n)=O(1)

3.显然，频度不同但时间复杂度可能相同，反之亦然。 比如：

T(4n2+4n+1)T(4n^2+4n+1)T(4n2+4n+1) 和 T(n2+2n+1)T(n^2+2n+1)T(n2+2n+1) 的 频度不同 ，但是它们的 时间复杂度 是相同的，都是 O(n2)O(n^2)O(n2).并且，大O符号的表示忽略低阶项和常数项，只保留最高次项。

4.如果一个算法的时间复杂度为 O(1)O(1)O(1)，O(log⁡n)O(\log n)O(logn)，O(n)O(n)O(n)，O(nlog⁡n)O(n \log n)O(nlogn)，O(nk)O(n^k)O(nk)，O(kn)O(k^n)O(kn) （其中 kkk 为常数 且 k>1k>1k>1），那么我们依次称这些算法具有：常数时间、对数时间、（幂）线性时间、线性对数时间、多项式时间、指数时间 。并且已按算法时间上的优劣排序，左边的时间比右边的时间要优。

注明：在分析时间复杂度的时候，认为 logloglog 不同的对数均为对数时间，且可认为均以 222 为底。且：

若 2k≤n<2k+12^k \leq n < 2^{k+1}2k≤n<2k+1 ，则 log⁡n=k\log n = klogn=k.

对数的增长很慢。即使 n=1020n=10^{20}n=1020 ，log⁡n\log nlogn 也只是 log⁡1020=66\log 10^{20} = 66log1020=66.

但是，一些初阶 oieroieroier 认为对数可以忽略不计，这是完全错误的。某些高等平衡树算法会达到 O(nlog⁡3n)O(n {\log}^3 n)O(nlog3n)，如果该算法控制不了常数，有时导致该平衡树算法与暴力 O(n2)O(n^2)O(n2) 相比没有什么优势。

分析时间复杂度的方法

理论上 ，一个程序准确的时间复杂度必须要运行才能得出。但因为大多数情况下我们不关心一些低阶项、或是常数项，只关心最高的次数（对数）。 比方说，一个 T(100n2)T(100n^2)T(100n2) 的算法时间仍被称为是 O(n2)O(n^2)O(n2) 的算法。

对于初阶选手，可以简单的认为： 有几重循环，最高次就是几。 但这仅仅适用于非常入门的选手。大多数情况这种方法都是错的。比方说下面一段程序：

#pragma GCC optimize(2)
#include<bits/stdc++.h>
using namespace std;

char s[22000010];
int p[22000010];
int n=0,ans=0;

inline void read(){
	char ch=getchar();
	s[0]='#'; s[n=1]='*';
	while(!isalpha(ch)) ch=getchar();
	while(isalpha(ch)) s[++n]=ch,s[++n]='*',ch=getchar();
}

int main(){
	read();
	int l,r=0,mid=0;
	for(l=1;l<=n;++l) {
		if(l<=r) p[l]=min(p[(mid<<1)-l],r-l+1);
		while(s[l-p[l]]==s[l+p[l]]) ++p[l];
		if(p[l]+l>r) r=p[l]+l-1,mid=l;
		ans=max(ans,p[l]);
	} printf("%d\n",--ans);
	return 0;
}

这是 mancher 算法 寻找字符串最长回文串的程序。

如果你是初阶选手，很有可能认为该时间复杂度为 O(n2)O(n^2)O(n2).

但实际上，该程序的时间复杂度是线性的，即O(n)O(n)O(n).

这样一个简单的例子我们了解到，分析时间复杂度不能一概而论，要分情况考虑。

下面了解一些算法或数据结构的时间复杂度。

一些字符串匹配的算法 （比如 KMP ，mancher ）全部都是线性的。

线段树维护一些值（可以区间合并的值，比如最大值、最小值、区间和、异或和、最大子段和等）的时间复杂度 一般来说 为 O(nlog⁡n+mlog⁡n)O(n \log n + m \log n)O(nlogn+mlogn).其中 nnn 为数组长度， mmm 为询问次数。

而二分的时间复杂度为 O(log⁡n)O(\log n)O(logn).其中 nnn 为二分的上限。当然大部分情况下 验证的时间复杂度为O(n)O(n)O(n)或O(n2)O(n^2)O(n2)，因此二分答案的时间复杂度很可能达到 O(nlog⁡n)O(n \log n)O(nlogn) 或是 O(n2)O(n^2)O(n2).

单调队列和单调栈的时间复杂度均为 O(n)O(n)O(n). 一般一个优秀的程序，for 里面套 while 都可以证明是线性的。（仅仅是一般）

还有一些高端可持久化线段树 或是 平衡树 、树套树之类的算法， 时间复杂度都可能达到 O(nlog⁡n)，O(nlog⁡2n)，O(nlog⁡3n)O(n \log n)，O(n {\log}^2 n)，O(n {\log}^3 n)O(nlogn)，O(nlog2n)，O(nlog3n) 等。

另外线性递推、矩阵等的算法，时间复杂度一般在 O(n2log⁡n)，O(n3)，O(n3log⁡n)O(n^2 \log n)，O(n^3)，O(n^3 \log n)O(n2logn)，O(n3)，O(n3logn) 这样的级别。（当然高斯消元也包括哦）

一些奇奇怪怪的斜率算法，时间复杂度不详。请读者自行考究。

至少分析一个程序的时间复杂度，要弄清每个变量、数组是干嘛的，结合经验等的来判断。

Case 2. 空间复杂度

没有什么“术语名词”可说的。把上面的 T 改成 S 就可以表示空间。当然 去掉低阶项和常数项的仍然表示为大O符号。 即：

S(4nn+4n+1)=O(n2)S(4n^n+4n+1)=O(n^2) S(4nn+4n+1)=O(n2)

但是，我们 必须 知道一些 存储单位：

  1. bit. 二进制数中的一个数位，即 000 或 111. 是计算机中数据的最小单位。（虽然用的很少，但是有些算法和它是有些关系的。比如 bitset 和 树状数组。）

  2. Byte，可缩写为 B，是字节，1B=8bit1B=8bit1B=8bit.

  3. KB，可缩写为 K，全称为 Kilobyte，是千字节，1KB=1024B1KB=1024B1KB=1024B.

  4. MB，可缩写为 M，全称为 Megabyte，是兆字节，1MB=1024KB1MB=1024KB1MB=1024KB.

  5. GB，可缩写为 G，全称为 Gigabyte，是吉字节，也是千兆，1GB=1024MB1GB=1024MB1GB=1024MB.

  6. TB，可缩写为 T，全称为 Terabyte，是万亿字节，也是太字节，1TB=1024GB1TB=1024GB1TB=1024GB.

后面的，可以 了解一下：

  1. PB，全称为 Petabyte，是千万亿字节，也是拍字节，1PB=1024TB1PB=1024TB1PB=1024TB.

  2. EB，全称为 Exabyte，是百亿亿字节，也是艾字节，1EB=1024PB1EB=1024PB1EB=1024PB.

  3. ZB，全称为 Zettabyte，是十万亿亿字节，也是泽字节，1ZB=1024EB1ZB=1024EB1ZB=1024EB.

  4. YB，全称为 Yottabyte，是一亿亿亿字节，也是尧字节，1YB=1024ZB1YB=1024ZB1YB=1024ZB.

  5. BB，全称为 Brontobyte，是一千亿亿亿字节，1BB=1024YB1BB=1024YB1BB=1024YB.

而 1024=2101024=2^{10}1024=210，所以 1BB=10249=2109=2901BB=1024^9={2^{10}}^9=2^{90}1BB=10249=2109=290.足足达到了 282828 位数，这是很令人惊异的。

不过 一般来说， 题目的内存限制都会以 MBMBMB 作单位。这是因为你 必须 知道：（下面用变量类型来代替该类型的变量）

  1. 一个 char 或者 一个 bool 均只占 111 个字节。

  2. 一个 short 或者一个 unsigned short 均占 222 个字节。

  3. 一个 int 或者一个 float 或者一个 long 或者一个 long int 或者一个 unsigned int 或者一个 ubsigned long ，均占 444 个字节。

  4. 一个 double 或者一个 long long 或者一个 long float，均占 888 个字节。

  5. 一个 long double ，占 161616 个字节。

事实上，如果我们要知道 long double 占的字节数，只需要：

printf("%d\n",sizeof(long double));

其中 sizeof 后面的括号里，就是一种变量类型。用这种方法可以计算上述所有的内存大小。

空间复杂度的计算

这比时间复杂度简单多了！ 在没有数组越界、指针访问非法内存等的情况下，一般来说 一个程序的内存只要看其开的容器的大小总和。这个容器包括 数组、STLSTLSTL、指针（指向容器）等等。 而单个的变量，或者常数级的变量，可以忽略不计。

比方说你先 int p[100000]，然后再 int a,b,c,d,e,f,g;

你想想，显然数组的开销比变量大得多啊！ 这和时间复杂度的计算有着异曲同工之妙。

具体的，有了存储单位和存储长度，做个乘法；把所有容器做个加法，大概就可以得到了。

Case 3. 如何优化算法复杂度

Case 3.1 时间复杂度

对于大部分情况，优化时间复杂度分为两类：

  • 常数上的优化

  • 算法（数据结构）上的优化

对于第一类，注意以下几条即可：

  1. 将 int 改为 register int（循环变量名中可使用）

  2. 在程序第一行添加 #pragma GCC optimize(2)

  3. 在所有函数前加 inline. 注明：有些人说该方法针对递归函数会出错，但笔者大量的实践表明，不会出错，只会更快。

  4. 将输入输出的 cin 或者 scanf 全部改为快读快输模板。

模板如下：

template <typename _tp> inline _tp read(_tp&x){
    char ch=getchar(),sgn=0;x=0;
    while(ch^'-'&&!isdigit(ch))ch=getchar();if(ch=='-')ch=getchar(),sgn=1;
    while(isdigit(ch))x=x*10+ch-'0',ch=getchar();if(sgn)x=-x;return x;
}

这是快读模板。然后有了这个函数，我们读入一个数只需要：

int x; read(x);

下面是一份输出模板

inline void write(int x) {
	if(x<10) {putchar(char(x%10+'0'));return;}
	write(x/10); putchar(char(x%10+'0'))
}

然后输出一个数（如果类型不是 int ，只要在函数开头更改）这样子：

write(x);

上述 444 个方法对大多数题目已经足够。如果对卡常数还有一定的兴趣，可以去参考一些资料：

  1. 了解一下 王逸松 这位选手，在 WC2017 的出题中，给了世界一道 卡常三合一，用百度百科的话，就是“让无数选手苦思冥想，并在其中提出了“松式基排”的概念。”甚至优秀的复杂度称为 O(wys)O(wys)O(wys).

并且，该题的第一个 Task 就要求给 2×1082 \times 10^82×108 个数进行排序，基数排序还一定要选对进制，不然就是一个 TLETLETLE...

  1. 了解一下成都七中的 nzhtl1477 巨佬，给出了 Ynoi 系列（不是云南 OIOIOI ，是由乃 OIOIOI ）。其中“以对常数与时间复杂度的严格要求、极高的难度、优美而冗长的题面以及与题面毫不相关的题目著称。大多使用数列分块算法，但也不乏对线段树、平衡树、计算几何等考点的考察。” Ynoi 系列 大都是紫题和黑题，这也是 nzhtl1477 成为了一名卡常级宗师。

而对于后者，常常需要重构代码。

一般的方法有：

  1. 用高级数据结构维护一些需要暴力的数组。（比如线段树、平衡树等等）

  2. 将需要不断尝试答案的算法，改为爬山，或者是二分、三分等。

  3. 对一些搜索，加以可行性剪枝。

  4. 如果是动态规划，那么尝试用改变状态来改变程序。（比如 LISLISLIS ）

  5. 套板子（比如快速幂），用高级算法（比如 KMPKMPKMP ）维护等。

  6. 对有单调性的东西可以直接队列上！

冗余方法不再赘述。

Case 3.2 空间复杂度

空间复杂度的优化其实没有必要，但是某种场合下还是需要的。

  1. 把冗余空间清除。

比如看这样一个例子：

给出 nnn 个数，求这 nnn 个数中前 kkk 小的 kkk 个数。

数据范围： n≤106n \leq 10^6n≤106，k≤103k \leq 10^3k≤103.

限制：时间 1000ms1000ms1000ms，空间 10MB10MB10MB.

你会发现如果直接存下来排序的话，空间是无法存下的。

所以，我们发现每次只要知道前 kkk 小的这 kkk 个数，所以数组只需要开 10310^3103 个记录。

对于一个新加的数，因为数组的记录是有序的，用二分找到它的位置插入即可；至于插入的维护方式，可以用链表实现。时间复杂度还是 O(nlog⁡n)O(n \log n)O(nlogn) ,但是空间复杂度从 O(n)O(n)O(n) 降到了 O(k)O(k)O(k).

  1. 滚动数组

这是一个常用的方法，尤其在 dp 的当中，如果你发现某个状态中的某个数，每次只决定于它前面那个状态的数，那么就可以滚动，一层层地覆盖。

其余的方法，因为现在还没有一个人专门卡空间，因此尚未得出其它结论。
